# Markdown Screenshot Checklist

Use these screenshots during the first-hour Markdown/Obsidian discussion. Do not turn this segment into a live editing demo.

## Required screenshots

1. **Raw Markdown source**
   - Show headings, bullets, task lists, and a fenced prompt block.

2. **Rendered Markdown**
   - Show the same content rendered clearly.

3. **OpenClaw context files**
   - Show Markdown-style context files at a high level: persona, memory, instructions, and workspace notes.
   - Do not show private content.

4. **Agent instruction file**
   - Show role, boundaries, expected output, and first task.

5. **Task list as work plan**
   - Show `- [ ]` tasks becoming a concrete agent work plan.

6. **Obsidian note view**
   - Show the same kind of Markdown note in Obsidian.
   - Emphasize that Obsidian helps humans organize durable notes.

7. **GitHub README rendering**
   - Show how Markdown becomes project documentation in GitHub.

## Safety rule for screenshots

Use sample content only. Do not screenshot private notes, secrets, API keys, personal data, client data, internal project names, or anything confidential.

## Suggested sample file

```markdown
# Workshop Memory

## Goal
Build a simple agent-assisted project plan.

## Agent instruction
You are a planning assistant. Ask three questions before proposing a plan.

## Task list
- [ ] Clarify the goal
- [ ] Identify constraints
- [ ] Propose next steps
```
