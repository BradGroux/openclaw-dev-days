# Facilitator Notes

## Purpose
These notes help map the agent roles to the live workshop flow so attendees can see not just what each role does, but when to use it.

## Suggested Timing For A 90-Minute Session

### Minutes 0-10 — Frame the Exercise
- introduce the startup idea
- explain the difference between one general AI assistant and specialized roles
- show the `agent-roles/README.md` overview

### Minutes 10-20 — Market Researcher
- use the Market Researcher to gather adjacent products and patterns
- discuss what feels familiar, crowded, or promising

### Minutes 20-30 — Idea Validator
- use the Idea Validator to cut scope and clarify the MVP
- challenge assumptions before building anything substantial

### Minutes 30-40 — Project Manager
- use the Project Manager to create the shortest path to a demo
- assign milestones and immediate next actions

### Minutes 40-75 — Coder
- use the Coder to scaffold and iterate on the MVP
- emphasize momentum, coherence, and demo readiness over completeness

### Minutes 75-85 — Outreach Assistant
- use the Outreach Assistant to explain the product clearly
- generate launch/demo/recap language

### Minutes 85-90 — Debrief
- review the outputs from each role
- talk about where specialization improved quality
- show how the workflow can be reused after the workshop

## Teaching Notes
- keep the example idea small enough to finish
- compare outputs from different attendees when possible
- show that agent specialization improves decision quality, not just speed
- remind people that prompts work better when the AI has a job, a boundary, and a deliverable

## Good Debrief Questions
- Which role created the most clarity?
- Where did the team try to skip steps?
- What changed when the idea got validated before coding?
- How did the messaging improve once the app actually existed?
