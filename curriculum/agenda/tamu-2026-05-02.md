# TAMU OpenClaw Workshop Agenda

**Event:** OpenClaw All-Day Hands-On Workshop  
**Date:** May 2, 2026  
**Time:** 9:00 AM - 5:00 PM  
**Hosts:** Brad Groux and Arnold Castro  
**Theme:** Build your startup team of AI agents and use them to bring **Beaver Badges** to life.

## Core Framing

This workshop is not just about installing OpenClaw.

It is about showing people how to:
- understand agentic AI in practical terms
- use Markdown memory as the foundation for continuity
- configure OpenClaw as a personal command center
- create a five-agent startup team
- orchestrate those agents to research, validate, plan, build, and promote a product
- leave with a system they can expand after the workshop

## Startup Team for the Day

Participants will build and work with this five-agent team:
- **Market Researcher**
- **Idea Validator**
- **Coder**
- **Project Manager**
- **Outreach Assistant**

Throughout the day, attendees will also learn the role of the **orchestrator**: the human-guided decision maker who sets direction, coordinates agents, and judges output quality.

## Product for the Day

The fictional startup we will build is **Beaver Badges**.

Beaver Badges is a fan-style web app inspired by Texas road trip culture. Users track store visits, unlock achievement badges, and brainstorm roadmap ideas such as maps, rankings, streaks, social sharing, and location-based expansion.

## Agenda

### 9:00-9:50 AM — Introduction, Agentic AI, and Memory

**Purpose:** Set the frame for the entire day.

**What we cover:**
- Welcome, introductions, and expectations
- What OpenClaw is and what it is not
- What AI agents are in plain English
- Why memory matters
- Why Markdown files are a strong foundation for AI continuity
- What a good orchestrator does
- How the five-agent startup team will work together
- Why building in public helps momentum and learning

**Key message:**
Agents are only useful when they have roles, memory, and direction. The human remains the orchestrator.

**Challenge before break:**
Post one thought, photo, or expectation from the event using **#TAMUopenclaw** on LinkedIn or Twitter.

---

### 10:00-11:30 AM — Prerequisites, Ollama, and First OpenClaw Success

**Purpose:** Get attendees to one working OpenClaw response before the room gets lost in optional integrations.

**What we cover:**
- Overview of the required stack
- Git and GitHub basics for people who are new
- Installing prerequisites on Mac and Windows
- Installing or confirming:
  - Visual Studio Code
  - Git
  - Obsidian
  - Ollama
  - OpenClaw
- Launching/configuring OpenClaw through Ollama
- Verifying the gateway and dashboard
- Overview of package managers:
  - Homebrew on Mac
  - Winget on Windows
- Helper lanes:
  - green: keep building
  - yellow: quick triage
  - red: rescue/demo path

**Operating principle:**
First success beats feature coverage. The default path is Ollama → OpenClaw gateway → dashboard → first response. Messaging and Azure are not blockers for this milestone.

**Challenge before lunch:**
Create a workspace folder, create your first Markdown memory file, and write three lines:
- what you want to build
- what kind of agent would help you most
- one thing you want to learn today

Then confirm:
- `ollama --version` works
- `openclaw --version` works
- `openclaw gateway status` works
- the dashboard opens
- your first prompt receives a response

### 12:15-1:50 PM — Provider Paths, Memory, and First Useful Operator Loop

**Purpose:** Move from “it responds” to “it helps.”

**What we cover:**
- Baseline provider path: Ollama/local or Ollama cloud depending on machine readiness
- Stretch provider path: Azure AI Foundry API key configuration
- Why Azure Foundry uses deployment names in OpenClaw: `microsoft-foundry/<deployment-name>`
- Key safety: do not paste API keys into shared docs, repos, screenshots, or projected chat
- Understanding sessions, memory, and channels
- First prompts to test the system
- Markdown memory loop
- Optional/demo-only mobile channel discussion after dashboard success

**Live demo sequence:**
1. Validate gateway/dashboard
2. Send first useful prompt
3. Create/read/update a memory note
4. Show memory-aware follow-up
5. Demo Azure Foundry configuration or smoke test if credentials are ready
6. Demo messaging/mobile channel only if the room is already green

**Challenge before break:**
Ask OpenClaw to summarize your workshop goal, suggest one agent you should create first, and save a useful note to memory.

---

### 2:00-2:50 PM — Build the Five-Agent Team

**Purpose:** Turn one assistant into a startup team.

**What we cover:**
- Why specialization matters
- Defining each of the five agents
- Example prompts, boundaries, and outputs for each role
- How the orchestrator decides what gets delegated
- How skills and memory make agent roles more useful over time

**Agent roles for the day:**
- **Market Researcher** → competitors, trends, market framing
- **Idea Validator** → assumptions, risks, user value, prioritization
- **Coder** → app structure, implementation, debugging, iteration
- **Project Manager** → scope, sequence, task breakdown, delivery checkpoints
- **Outreach Assistant** → social copy, launch ideas, positioning, feedback requests

**Challenge before next block:**
Define your own version of the five-agent team and give each agent one sentence of responsibility.

---

### 3:00-3:50 PM — Build Beaver Badges

**Purpose:** Show how the team works on a real product.

**What we cover:**
- Introduce the Beaver Badges concept
- Use the Market Researcher to gather comparable ideas
- Use the Idea Validator to sharpen the MVP
- Use the Project Manager to define scope for the session
- Use the Coder to scaffold a React + Vite + Tailwind app
- Use the Outreach Assistant to prepare launch-style messaging
- Work with a JSON-backed data model instead of a database

**Core MVP for the live build:**
- location list
- visited/unvisited state
- badge milestone logic
- progress view
- simple achievement display

**Roadmap topics to discuss live:**
- map support
- rankings
- streaks
- reviews
- trip planning
- community features

**Challenge before final hour:**
Have your agents propose one feature to add next and justify whether it belongs in the MVP or roadmap.

---

### 4:00-5:00 PM — Showcase the Possible

**Purpose:** End with momentum, not just completion.

**What we cover:**
- Demonstrate how Beaver Badges can grow from an MVP into a roadmap
- Show how Veritas Kanban fits into agentic execution
- Show how a command center can evolve over time
- Explain how to keep building after the event
- Encourage open source contribution and building in public
- Introduce 30-day follow-through expectations
- Mention SSTB community path, Discord, and future support options

**Closing challenge:**
Commit to a 30-day build goal:
- keep improving Beaver Badges, or
- build your own product using your startup team of agents

**Final message:**
Do not aim for impressive. Aim for real, finished, and forward-moving.
