# OpenClaw Dev Days Curriculum

This folder contains the reusable teaching path for OpenClaw Dev Days.

The curriculum is built around a simple progression:

1. Get OpenClaw running.
2. Create durable Markdown context.
3. Give agents clear roles and boundaries.
4. Test agent output against an explicit contract.
5. Orchestrate a small team through a working loop.
6. Control context, cost, risk, and handoff.

## Facilitator Stance

Protect the first working loop. The deeper material is valuable only after attendees have seen OpenClaw respond, used a Markdown memory file, and understood that the human remains the orchestrator.
