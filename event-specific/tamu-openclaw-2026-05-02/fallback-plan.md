# TAMU Fallback Plan

Use this if installs, network, accounts, or provider setup start consuming the room.

## Rule

Protect momentum. Do not let setup problems swallow the workshop.

If a blocker takes more than 10 minutes, move the attendee to observe, pair, or use the rescue/demo path.

## If Wi-Fi is unreliable

- Keep the presenter machine connected if possible.
- Move attendees into observation/pairing mode.
- Use local screenshots and prepared repo materials.
- Focus on concepts, prompts, Markdown files, and agent workflow.
- Resume hands-on setup when connectivity stabilizes.

## If installs fail

- Put the attendee in the red lane.
- Pair them with someone who is already running.
- Use the presenter machine for the next live demo.
- Keep the attendee following along in the repo and notes.
- Revisit their machine during lunch or helper time.

## If Ollama/OpenClaw setup fails

Use this order:

1. Confirm basic commands.
2. Confirm Ollama is running.
3. Confirm OpenClaw launched.
4. Confirm dashboard/gateway access.
5. If still blocked, move to rescue/demo path.

Do not spend the group block debugging one machine.

## If Azure Foundry setup fails

- Keep attendees on Ollama.
- Mark Azure as post-first-success support.
- Return to Azure only after the room is stable.
- Do not let provider setup block the core OpenClaw loop.

## If the Beaver Badges app fails live

- Use the README and PRD as the teaching artifact.
- Walk through how the agent team would improve the project.
- Use the README improvement exercise.
- Show the intended app flow from code/data if needed.
- Preserve the product/team/orchestration lesson even without a live app.

## If everything breaks

Run the workshop as a guided walkthrough:

1. Explain OpenClaw and agents.
2. Show Markdown/context screenshots.
3. Walk through the repo structure.
4. Use prompts verbally and in Markdown files.
5. Show the demo machine if available.
6. Have attendees write their own `workshop-memory.md` and `agent-plan.md` locally.
7. End with a 30-day build goal and community follow-up.

## Minimum viable workshop outcome

Even if setup fails, attendees should leave with:

- a clear understanding of agents and orchestration
- a Markdown memory file
- an agent instruction
- a task list
- a Beaver Badges product walkthrough
- a next step for continuing after the event
