# First-Hour Speaker Notes

These are quick presenter beats for the opening hour. Keep this segment conceptual and visual. Do not turn it into a live Markdown editing lab.

## Opening frame

- Today is about getting one real operator win.
- The goal is not to memorize commands.
- The goal is to understand how people, agents, memory, tools, and review work together.

## What OpenClaw is

- OpenClaw is a local/operator-oriented AI workspace.
- It can chat, use tools, keep context, and support agent workflows.
- It becomes useful when it has clear instructions, useful memory, and bounded tasks.

## What agents are

- Agents are not magic employees.
- They are role-based helpers working from context, instructions, tools, and feedback.
- Good agents need:
  - a role
  - a goal
  - useful context
  - boundaries
  - expected output
  - human review

## Models use context

- Models do not remember like people do.
- They work from the context they are given.
- If the context is messy, vague, or missing, the output gets worse.
- If the context is structured, explicit, and durable, the output gets better.

## Markdown is the thinking layer

Key line:

> Markdown is how we make thinking legible to both people and agents.

Explain:

- Markdown is plain text with structure.
- Humans can read it.
- Agents can parse it.
- GitHub can render it.
- Obsidian can organize it.
- OpenClaw can use it as durable context.

## OpenClaw Markdown files

Talk through the idea of Markdown files as context:

- identity/persona
- user preferences
- memory
- workspace notes
- agent instructions
- project plans
- task lists

Do not show private or sensitive content. Use sample/demo content only.

## Obsidian connection

- Obsidian is not required for an agent to read Markdown.
- Obsidian helps humans manage Markdown over time.
- It turns notes into a personal knowledge base.
- It helps keep agent-created data from disappearing into chat history.

## GitHub connection

- GitHub uses Markdown for READMEs, issues, PRs, discussions, and documentation.
- A good README helps both people and agents understand a project.
- The Beaver Badges README becomes an early improvement task later in the day.

## Safety checkpoint

Before GitHub or shared files, say:

> Do not put secrets, API keys, private data, sensitive personal notes, or anything confidential in a public repository, shared file, screenshot, or projected prompt. When in doubt, keep it local.

## Transition to setup

Close the first hour by saying:

- Now that the model/agent/context idea is clear, the next goal is simple.
- Get the tools installed.
- Get one OpenClaw response.
- Then use Markdown to turn that first response into durable context.
